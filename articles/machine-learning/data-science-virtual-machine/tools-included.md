# <a name="what-tools-are-included-on-the-azure-data-science-virtual-machine"></a>Jaké nástroje jsou součástí virtuálního počítače Azure Data Science?

Virtuální počítač pro datové vědy je nejjednodušší způsob, jak prozkoumat data a dělat strojové učení v cloudu. Virtuální počítače pro datové vědy jsou předem nakonfigurovány s kompletním operačním systémem, opravami zabezpečení, ovladači a oblíbeným softwarem pro datovou vědu a vývoj. Můžete si vybrat hardwarové prostředí, od levnějších počítačů zaměřených na procesor až po velmi výkonné počítače s více grafickými procesory, úložištěm NVMe a velkým množstvím paměti. U počítačů s grafickými procesory jsou nainstalovány všechny ovladače, všechny architektury strojového učení jsou přizpůsobeny kompatibilitě gpu a zrychlení je povoleno ve všech aplikačních software, který podporuje gpu.

Virtuální stroj pro datové vědy je dodáván s nejužitečnějšími nástroji pro datové vědy, které jsou předinstalovány:

## <a name="how-to-choosecondaenvironment"></a>Jak si vybrat conda prostředí  

> [!NOTE]
> Při přepnutí na novou verzi virtuálního počítače pro datové vědy ze staré verze dochází k nejnovějším změnám. 

**Přechod z Windows Serveru 2016 na Windows Server 2019** </br> 

* Prostředí conda je nastaveno na python37_default. Nepodporujeme python2. Pokud jste používali tensorflow nebo pytorch v prostředí python conda, použijte py37_tensorflow nebo py37_pytorch. Pokud jste používali jenom AzureML, použijte azureml_py36_automl </br>

* Pokud jste používali tensorflow nebo pytorch v prostředí AzureML conda, použijte azureml_py36_tensorflow nebo azureml_py36_pytorch. </br>

**Přechod z Ubuntu 16.04 na Ubuntu 18.04** </br>

* Pokud jste používali tensorflow nebo pytorch ve vašem prostředí azureml_py36 conda, použijte azureml_py36_tensorflow nebo azureml_py36_pytorch. </br>

* Pokud jste používali tensorflow nebo pytorch v prostředí py37 conda, použijte py37_tensorflow nebo py37_pytorch. </br>

* V opačném případě použijte python37_default nebo pokud jste používali pouze azureml_py36 azureml_py36_automl </br>
