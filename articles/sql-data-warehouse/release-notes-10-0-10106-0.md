# <a name="azure-sql-data-warehouse-release-notes"></a>Poznámky k verzi Azure SQL Data Warehouse

Tento článek shrnuje nové funkce a vylepšení v posledních verzích [Azure SQL Data Warehouse](sql-data-warehouse-overview-what-is.md) (Azure SQL DW). V tomto článku jsou uvedeny také významné aktualizace obsahu, které přímo nesouvisejí s vydáním, ale publikované ve stejném časovém rámci. Vylepšení dalších služeb Azure najdete v tématu [aktualizace služby](https://azure.microsoft.com/updates).

## <a name="check-your-azure-sql-data-warehouse-version"></a>Ověřit verzi Azure SQL Data Warehouse

Protože se nové funkce zavádějí do všech oblastí, podívejte se na verzi, která je nasazená do vaší instance, a nejnovější poznámky k verzi Azure SQL DW pro dostupnost funkcí. Pokud chcete zjistit verzi Azure SQL DW, připojte se k datovému skladu pomocí SQL Server Management Studio (SSMS) a spusťte `SELECT @@VERSION;` a vraťte aktuální verzi Azure SQL DW.

Příklad výstupu:

![Verze SQL Data Warehouse](./media/release-notes/sql_data_warehouse_version.png)

Pomocí zjištěného data ověřte, která verze se použila pro Azure SQL DW.

Úložiště dotazů je sada interních úložišť a zobrazení dynamické správy (zobrazení dynamické správy), která umožňují:<br/><br/>&bull; &nbsp; identifikovat a ladit nejlepší dotazy pro využívání prostředků<br/>&bull; &nbsp; určení a vylepšení neplánovaných úloh<br/>&bull; &nbsp; vyhodnotit výkon dotazů a ovlivnit plán změnami v statistikách, indexech nebo systémové velikosti (nastavení DWU)<br/>&bull; &nbsp; zobrazení úplného textu dotazu pro všechny spuštěné dotazy<br/><br/>Úložiště dotazů obsahuje tři skutečné obchody:<br/>&bull; &nbsp; úložiště plánů pro uchování informací o plánu provádění<br/>&bull; &nbsp; úložiště statistik modulu runtime pro uchování informací o statistice spouštění<br/>&bull; &nbsp; úložiště statistiky čekání na trvalé informace o statistikách čekání.<br/><br/>SQL Data Warehouse tyto obchody spravuje automaticky a poskytuje neomezený počet dotazů storied za posledních sedm dnů bez dalších poplatků. Povolení úložiště dotazů je jednoduché jako spuštění příkazu ALTER DATABASE jazyka T-SQL: <br/>SQL----ALTER DATABASE [DatabaseName] SET QUERY_STORE = ON;-------Další informace o úložišti dotazů v Azure SQL Data Warehouse naleznete v článku, [monitorování výkonu pomocí úložiště dotazů](/sql/relational-databases/performance/monitoring-performance-by-using-the-query-store)a zobrazení dynamické správy úložiště dotazů, jako je například [Sys. query_store_ dotaz](/sql/relational-databases/system-catalog-views/sys-query-store-query-transact-sql). Tady je [Blogový příspěvek](https://azure.microsoft.com/blog/automatic-performance-monitoring-in-azure-sql-data-warehouse-with-query-store/) , který oznamuje vydání verze.|
|**Nižší výpočetní vrstvy pro Azure SQL Data Warehouse Gen2**|Azure SQL Data Warehouse Gen2 nyní podporuje nižší výpočetní úrovně. Zákazníci můžou vyskytnout Azure SQL Data Warehouse výkon, flexibilitu a funkce zabezpečení, počínaje 100 cDWU ([jednotky datového skladu](what-is-a-data-warehouse-unit-dwu-cdwu.md)) a škálovat na 30 000 cDWU během několika minut. Od prosince 2018 můžou zákazníci využívat Gen2 výkon a flexibilitu s nižšími výpočetními úrovněmi v [oblastech](gen2-migration-schedule.md#automated-schedule-and-region-availability-table), přičemž zbývající oblasti jsou dostupné během 2019.<br/><br/>Vyřazením vstupního bodu pro novou generaci datových skladů otevře Microsoft dveře pro zákazníky s hodnotami, kteří chtějí vyhodnotit všechny výhody zabezpečeného a vysoce výkonného datového skladu bez odhadu toho, které zkušební prostředí je pro ně nejvhodnější. Zákazníci můžou začít s nízkým množstvím 100 cDWU, a to od aktuálního cDWU vstupního bodu 500. SQL Data Warehouse Gen2 nadále podporuje operace pozastavení a obnovení a překračuje jenom flexibilitu ve výpočetním prostředí. Gen2 také podporuje neomezenou úložnou kapacitu úložiště ve sloupcích spolu s 2,5 časy více paměti na dotaz, až 128 souběžných dotazů a [adaptivních funkcí ukládání do mezipaměti](https://azure.microsoft.com/blog/adaptive-caching-powers-azure-sql-data-warehouse-performance-gains/) . Tyto funkce v průměru přinášejí více výkonu v porovnání se stejnou jednotkou datového skladu na Gen1 za stejnou cenu. Geograficky redundantní zálohy jsou standard pro Gen2 s integrovanou zaručenou ochranou dat. Azure SQL Data Warehouse Gen2 je připravené škálovat, když jste.|
|**Sloučení na pozadí columnstore**|Ve výchozím nastavení Azure SQL Data Warehouse (Azure SQL DW) ukládá data ve sloupcovém formátu s použitím mikrooddílů s názvem [rowgroups](sql-data-warehouse-memory-optimizations-for-columnstore-compression.md). V důsledku omezení paměti při sestavení indexu nebo při načítání dat se může rowgroups komprimovat s méně než optimální velikostí 1 000 000 řádků. Rowgroups se může také fragmentovat z důvodu odstranění. Malé nebo fragmentované rowgroupsy mají za následek vyšší spotřebu paměti a také neefektivní provádění dotazů. V této verzi Azure SQL DW úloha údržby na pozadí columnstore sloučí malé komprimované rowgroups a vytvoří větší rowgroups, aby lépe využila paměť a urychlila spouštění dotazů.
