# <a name="azure-cognitive-services-support-and-help-options"></a>Možnosti podpory a nápovědy azure cognitive services

Teprve začínáte zkoumat funkce služby Azure Cognitive Services? Implementujete novou funkci ve vaší aplikaci? Zde jsou návrhy, kde můžete získat pomoc jako vývojář.

> [!div class="checklist"]
> * Buďte informováni o novém vývoji ve *službách Azure Cognitive Services*.
> * Poznámky k verzi obsahují informace pro všechny verze.
> * Vyhledejte, zda váš problém byl projednán komunitou nebo zda již existuje existující dokumentace.
> * Pokud nemůžete najít uspokojivou odpověď, zeptejte se na *Stack Overflow*.
> * Pokud najdete problém s jedním z ukázek na GitHubu, vyvoláte problém *GitHubu.*
> * Vyhledejte řešení ve *fóru UserVoice*.

## <a name="create-an-azure-support-request"></a>Vytvoření žádosti o podporu Azure

<div class='icon is-large'>
    <img alt='Azure support' src='https://docs.microsoft.com/media/logos/logo_azure.svg'>
</div>

Prozkoumejte celou řadu [možností podpory Azure a vyberte si plán,](https://azure.microsoft.com/support/plans) který nejlépe vyhovuje, ať už jste vývojář, který právě začíná vaši cestu do cloudu, nebo velká organizace, která nasazuje strategické aplikace, které jsou důležité pro podnikání. Zákazníci Azure můžou vytvářet a spravovat žádosti o podporu na webu Azure Portal.

* [Portál Azure](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview)
* [Portál Azure pro vládu Spojených států](https://portal.azure.us)

## <a name="post-a-question-on-stack-overflow"></a>Zaúčtování otázky k přetečení zásobníku

<div class='icon is-large'>
    <img alt='Stack Overflow' src='https://docs.microsoft.com/media/logos/logo_stackoverflow.svg'>
</div>

Okamžité odpovědi od komunity najdete v části Přetečení zásobníku. Přetečení zásobníku je upřednostňovaný kanál pro otázky související s vývojem. Je to místo, kde členové komunity a členové týmu Microsoft jsou přímo zapojeni do pomoci při řešení vašich problémů.

Pokud nemůžete najít odpověď na váš problém prostřednictvím vyhledávání, odešlete novou otázku do zásobníku Přetečení. Při formulování otázky použijte jednu z následujících značek:

 - [Microsoft Kognitivní](https://stackoverflow.com/questions/tagged/microsoft-cognitive)
 - [Azure Cognitive Services](https://stackoverflow.com/questions/tagged/azure-cognitive-services)

> [!TIP]
> Následující příspěvky z přetečení zásobníku obsahují tipy, jak vytvářet otázky a přidávat zdrojový kód. Následující pokyny mohou pomoci zvýšit pravděpodobnost, že členové komunity vaši otázku rychle vyhodnotí a na vaši otázku odpoví:
> * [Jak se mám zeptat na dobrou otázku?](https://stackoverflow.com/help/how-to-ask)
> * [Jak vytvořit minimální, reprodukovatelný příklad?](https://stackoverflow.com/help/minimal-reproducible-example)

## <a name="create-a-github-issue"></a>Vytvoření problému githubu

<div class='icon is-large'>
    <img alt='GitHub' src='https://docs.microsoft.com/media/logos/logo_octokitty.svg'>
</div>

Ukázky jsou často účtovány jako open source. V případě dotazů a problémů vytvořte *problém* v příslušném úložišti GitHub. Můžete odeslat žádost o přijetí k námzetí taky! Hledáte zdrojový kód **Azure Samples,** níže je seznam ukázkový zdrojový kód služeb Cognitive Services:

* [Ukázky Azure – kognitivní služby](https://github.com/Azure-Samples?utf8=%E2%9C%93&q=cognitive&type=&language=)

Můžete vytvořit hlášení o chybě, žádost o funkci nebo položit obecnou otázku a sdílet osvědčené postupy. Pro hlášení chyb postupujte podle dodaných šablon:

**Popište chybu**

Jasný a stručný popis toho, co je chyba.

**Chcete-li reprodukovat**

Kroky pro reprodukci chování:
1. ...
2. ...

**Očekávané chování**

Jasný a stručný popis toho, co jste očekávali, že se stane.

**Které nabídky služeb Cognitive Services používáte**

Podrobně upřesněte nabídku služeb Cognitive Services, kterou se pokoušíte použít, a uveďte podrobnosti o tom, čeho se pokoušíte dosáhnout.

**Platforma, operační systém a programovací jazyk**

 - OS: [např.
 - Hardware - x64, x86, ARM, ...
 - Prohlížeč, například Chrome, Safari] (je-li k dispozici)- buďte

**Kontext**

 - Chybové zprávy, informace protokolu, trasování zásobníku, ...
 - Pokud nahlásíte chybu pro konkrétní interakci služby, nahlaste SessionId a čas (včetně časového pásma) nahlášených incidentů. SessionId je hlášena ve všech zpětné volání/události, které obdržíte.
 - Jakékoli další další informace
