# <a name="different-stages-in-mobile-application-development"></a>Různé fáze vývoje mobilních aplikací
Sestavování mobilní aplikace zahrnuje několik fází a vývojáři můžou snadno využít výhod služeb, nástrojů a technologií Microsoftu k vytváření vysoce kvalitních mobilních aplikací.

## <a name="app-development"></a>Vývoj aplikací
Vývojáři mohou vytvářet nativní klientské aplikace s jednou platformou s použitím jazyků, jako jsou například cíle C a Java, aplikace pro různé platformy pomocí Xamarin, reakce nativních, Unity a hybridních aplikací pomocí Cordova. Můžou využít editory pro IDEs a kódování, jako je Visual Studio, Visual Studio pro Mac nebo Visual Studio Code na počítačích PC a Mac, a vytvořit tak klientské aplikace.

## <a name="plan-and-design-with-back-end-services"></a>Plánování a návrh pomocí back-endové služby
 Microsoft poskytuje celou řadu služeb pro nastavení zabezpečeného, škálovatelného a spolehlivého back-endu pro vaši mobilní aplikaci. Vývojáři mohou vytvářet rozhraní API a triggery bez serveru, aniž by museli udržovat své vlastní servery. Můžou využívat služby, jako jsou nabízená oznámení, synchronizace offline dat, ověřování a data, která dohromady tvoří komplexní a obsáhlém mobilní back-end jako službu, která podporuje nejrůznější scénáře vytváření aplikací. Sestavení back-end služby má mnoho požadavků:
   - Omezí ukládání dat na zařízení.
   - Synchronizujte data napříč více zařízeními.
   - Zajištění spuštění aplikace bez připojení k síti.
   - Odesílání oznámení uživatelům.
   - Snižte spotřebu baterie.
   - Vytváření škálovatelných, spolehlivých a zabezpečených back-endu.
   - Automatické maintainence serveru.

## <a name="devops-and-continuous-monitoring"></a>DevOps a nepřetržité monitorování
Vývojáři vytvářející mobilní a cloudové aplikace můžou využít výhod průběžné integrace (CI), průběžného doručování (CD) a nepřetržité monitorovací služby pro vytváření nativních sestavení pro všechny platformy. testy automatizace uživatelského rozhraní v tisících fyzická zařízení, Automatizujte jejich vydávání prostřednictvím testovacích kanálů beta verze nebo nasaďte přímo do obchodů s aplikacemi. Kromě toho mohou pro své aplikace shromažďovat informace o analýze, chybách a výjimkách využití.

## <a name="additional-services"></a>Další služby
Existuje mnoho služeb Azure, které je možné spotřebovat v závislosti na typu a charakteru sestavené aplikace.
  - Azure Kognitivní hledání pro hledání všech typů interních dat s rychlým dotazování.
  - Azure Cognitive Services pro použití umělých inteligentních funkcí ve vaší aplikaci.
  - Signál Azure pro povolení komunikace v reálném čase ve vaší aplikaci.
