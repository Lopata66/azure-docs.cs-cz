# <a name="azure-synapse-analytics-release-notes"></a>Poznámky k verzi Azure Synapse Analytics

Tento článek shrnuje nové funkce a vylepšení v posledních verzích [Synapse SQL](sql-data-warehouse-overview-what-is.md) v Azure Synapse. Článek také uvádí významné aktualizace obsahu, které přímo nesouvisejí s vydáním, ale jsou publikovány ve stejném časovém rámci. Vylepšení dalších služeb Azure najdete v [tématu Aktualizace služeb](https://azure.microsoft.com/updates).

## <a name="check-your-azure-synapse-version"></a>Zkontrolujte verzi Azure Synapse

Jak se nové funkce zavádějí do všech oblastí, zkontrolujte verzi nasazenou do vaší instance a nejnovější poznámky k verzi dostupnosti funkcí. Chcete-li zkontrolovat verzi, připojte se k fondu SQL prostřednictvím `SELECT @@VERSION;` SQL Server Management Studio (SSMS) a spusťte vrátit aktuální verzi.

Pomocí identifikované verze potvrďte, která verze byla použita ve vašem fondu SQL. Datum ve výstupu identifikuje měsíc pro vydání použité pro fond SQL.

> [!NOTE]
> Název produktu vrácený@VERSION službou SELECT @ se změní z datového skladu Microsoft Azure SQL na Azure Synapse Analytics. Před změnou vám zašleme předběžné oznámení. Tato změna je relevantní pro zákazníky, kteří analyzovat@VERSION název produktu z výsledku SELECT @ v jejich kódu aplikace. Chcete-li se vyhnout změnám kódu aplikace v důsledku změny značky produktu, použijte tyto příkazy k dotazování serverproperty pro název produktu databáze a verze: Chcete-li vrátit číslo verze XX. X.XXXXX. X (bez názvu produktu) použijte tento příkaz:
>
> ```sql
> SELECT SERVERPROPERTY('ProductVersion')
>
> --To return engine edition, use this command that returns 6 for Azure Synapse Analytics (Formerly SQL Data Warehouse):
>
> SELECT SERVERPROPERTY('EngineEdition')
> ```

Zákazníci mohou začít tak nízké, jak 100 cDWU, dolů od aktuálního 500 cDWU vstupní bod. SQL Analytics nadále podporuje pozastavit a obnovit operace a přesahuje pouze flexibilitu ve výpočetních prostředcích. Gen2 také podporuje neomezenou kapacitu úložiště sloupců spolu s 2,5krát více paměti na dotaz, až 128 souběžných dotazů a [funkce adaptivního ukládání do mezipaměti.](https://azure.microsoft.com/blog/adaptive-caching-powers-azure-sql-data-warehouse-performance-gains/) Tyto funkce v průměru přinášejí pětkrát vyšší výkon ve srovnání se stejnou jednotkou datového skladu na Gen1 za stejnou cenu. Geograficky redundantní zálohy jsou standardem pro Gen2 s vestavěnou zaručenou ochranou dat. SQL Analytics v Azure Synapse je připravena škálovat, když jste.|
|**Sloučení pozadí columnstore**|Ve výchozím nastavení azure SQL data ukládá data ve sloupcovém formátu, s mikrooddíly s názvem [rowgroups](sql-data-warehouse-memory-optimizations-for-columnstore-compression.md). Někdy z důvodu omezení paměti při vytváření indexu nebo době načítání dat mohou být skupiny řádků komprimovány s menší než optimální velikost jednoho milionu řádků. Rowgroups může být také fragmentován z důvodu odstranění. Malé nebo fragmentované rowgroups za následek vyšší spotřebu paměti, stejně jako neefektivní provádění dotazů. V této verzi úloha údržby na pozadí columnstore slučuje malé komprimované rowgroups a vytváří větší skupiny řádků, aby lépe využívaly paměť a urychlily provádění dotazů.
