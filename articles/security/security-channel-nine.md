# <a name="azure-security-videos-on-channel-9"></a>Videa o zabezpečení Azure na webu Channel 9
[Channel 9](https://channel9.msdn.com/) je komunita, která přináší vpřed prezentacím naše produkty a připojí se zákazníky.

Uvažují v softwaru je velká budoucnost, a jsou to nadšení. Channel 9 je komunita k účasti v probíhající konverzace.

Následující je uspořádaný seznam zabezpečení Azure prezentací na webu Channel 9. Nezapomeňte zaškrtnout každý měsíc o nových videích na této stránce.

[Urychlení konektoru Azure Consumption se zabezpečením Barracuda](https://channel9.msdn.com/events/Microsoft-Azure-Marketplace-ISV-Solutions-Webinar-Series/Webinar-1-Accelerating-Azure-Consumption-with-Barracuda-Security/Webinar-1-Accelerating-Azure-Consumption-with-Barracuda-Security)

Zobrazit, jak můžete Barracuda zabezpečení k zabezpečení vašich nasazení Azure.

[Azure Security Center – detekce hrozeb](https://channel9.msdn.com/Shows/Azure-Friday/Azure-Security-Center-Threat-Detection)

Pomocí Azure Security Center získáte přehled o stavu zabezpečení všech vašich prostředků Azure. Můžete rychle ověřit patřičnou úroveň a správnou konfiguraci kontrolních mechanismů zabezpečení. Scott hovoří s blatníků Sarah, který popisuje, jak Security Center umožňuje integrací detekce hrozeb.

[Přehled služby Azure Security Center](https://channel9.msdn.com/Shows/Azure-Friday/Azure-Security-Center-Overview)

Pomocí Azure Security Center získáte přehled o stavu zabezpečení všech vašich prostředků Azure. Můžete rychle ověřit patřičnou úroveň a správnou konfiguraci kontrolních mechanismů zabezpečení. Scott hovoří s blatníků Sara, která vysvětluje, to všechno!

[Živá ukázka: Ochrana proti, zjišťování a reakce na hrozby](https://channel9.msdn.com/events/Virtual-Security-Summit/Virtual-Security-Summit-2016/Live-Demo-Protecting-against-Detecting-and-Responding-to-Threats)

Připojte se k této relaci zobrazíte platformu Microsoft security v akci. Generální ředitel pro Cloudy a podniky, Julii White, ukazuje funkce zabezpečení Windows 10, Azure a Office 365, který může pomoci zabezpečit vaši organizaci.

[Šifrování v SQL Server Virtual Machines v Azure pro lepší zabezpečení](https://channel9.msdn.com/Shows/Azure-Friday/Encryption-in-SQL-Azure-for-better-security)

Jan Richins vás naučí [Scott](https://channel9.msdn.com/Niners/Glucose) snadno zašifrování jeho databází systému SQL Server na Azure Virtual Machine instances. Je to snazší, než jste si mysleli!

Oblasti jsou zahrnuté v tomto videu:

* Principy šifrování a systému SQL Server
* Informace k Data Protection API, hlavní klíče a certifikáty
* K vytvoření hlavního klíče a certifikáty a šifrování databáze pomocí příkazů SQL

[Jak nastavit zabezpečení ve službě DevTest Labs](https://channel9.msdn.com/Blogs/Azure/How-to-set-security-in-your-DevTest-Lab)

Jako vlastník testovacího prostředí se dají zabezpečit laboratoře přístupný přes dvě role prostředí: Vlastníka a uživatel služby DevTest Labs. Osoba v roli vlastník má úplný přístup v testovacím prostředí, zatímco uživatel v roli uživatel služby DevTest Labs má omezený přístup. V tomto videu si ukážeme, jak přidat uživatele do testovacího prostředí v některém z těchto rolí.

[Správa tajných kódů pro aplikace Azure](https://channel9.msdn.com/events/Build/2016/P456)

Každá vážné aplikace, které můžete nasadit v Azure má důležitých tajných kódů – připojovací řetězce, certifikáty, klíče. I chyby při správě těchto tajných kódů vede k závažné důsledky – nevracení, výpadky, narušení dodržování předpisů. Protože více poslední zjišťování zdůraznit, i chyby způsobují další čtyřnásobnou úniky dat než nežádoucí osoby. V této relaci budeme za některé osvědčené postupy pro správu tajných kódů důležitých aplikací. Těchto osvědčených postupů může jevit jako zdravý, ale je setkávat celá řada vývojářů. Budeme také popisuje, jak používat Azure Key Vault k implementaci těmito osvědčenými postupy. Jako dodatečná výhoda následující pomůže tyto postupy můžete prokázat dodržování standardů, jako je například SOC. Prvních 10 minut relace jsou na úrovni 100 a se vztahují ke všem cloudovým aplikacím, které můžete vyvíjet na jakékoli platformě. Zbývající úrovně 200 300 a se zaměřuje na aplikace, které vytvoříte na platformě Azure.

[Azure AD Privileged Identity Management: Průvodce, výstrahy, revize zabezpečení](https://channel9.msdn.com/Series/Azure-Active-Directory-Videos-Demos/Azure-AD-Privileged-Identity-Management-Security-Wizard-Alerts-Reviews)

Azure Active Directory (AD) Privileged Identity Management je funkce úrovně premium, která umožňuje zjišťovat, omezovat a monitorovat privilegované identity a jejich přístup k prostředkům. Vynucuje také na vyžádání, právě přístup správce v případě potřeby. Další informace o:

* Správa ochrany pro role pro správu Office 365 specifické úlohy
* Konfigurace Azure Multi-Factor Authentication Authentication(MFA) pro aktivace privilegovaných rolí
* Měření a zlepšit tak stav zabezpečení tenanta
* Sledování a opravy zabezpečení zjištění
* Kontrola, kdo potřebuje zůstat v privilegovaných rolích pro pracovní postupy pravidelné vydal
