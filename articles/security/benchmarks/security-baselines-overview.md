# <a name="overview-of-azure-security-baselines"></a>Přehled směrných plánů zabezpečení Azure

Azure Security Baselines vám pomůže posílit zabezpečení našich produktů díky vylepšeným funkcím nástrojů, sledování a zabezpečení a poskytují konzistentní prostředí při zabezpečení vašeho prostředí.

Směrné plány zabezpečení služby Azure se zaměřují na oblasti řízení zaměřené na cloud. Tyto ovládací prvky jsou v souladu s dobře známými standardy zabezpečení, například těmi, které popisuje Centrum pro zabezpečení Internetu (CIS). Naše směrné plány poskytují pokyny pro kontrolní oblasti uvedené v [benchmarku zabezpečení Azure](overview.md).

Každé doporučení obsahuje následující informace:
- **Azure ID**: Azure Security Benchmark ID, které odpovídá doporučení.
- **Doporučení**: Následující bezprostředně po Azure ID, doporučení poskytuje popis na vysoké úrovni ovládacího prvku.
- **Pokyny**: Odůvodnění doporučení a odkazy na pokyny, jak jej provést. Pokud je doporučení podporováno Službou zabezpečení Azure, budou tyto informace také uvedeny.
- **Odpovědnost**: Kdo je zodpovědný za provádění ovládacího prvku. Možnými scénáři jsou odpovědnost zákazníka, odpovědnost společnosti Microsoft nebo sdílená odpovědnost.
- **Monitorování Azure Security Center:** Určuje, zda ovládací prvek monitoruje Azure Security Center s odkazem na odkaz.

Všechna doporučení, včetně doporučení, která se nevztahují na tuto konkrétní službu, jsou zahrnuty ve směrné plánu, aby vám poskytly úplný přehled o tom, jak azure benchmark zabezpečení se vztahuje ke každé službě. Občas mohou existovat ovládací prvky, které nejsou použitelné z různých důvodů – například ovládací prvky Zaměřené na IaaS/výpočetní prostředky (například ovládací prvky specifické pro správu konfigurace operačního systému) nemusí být použitelné pro služby PaaS.
