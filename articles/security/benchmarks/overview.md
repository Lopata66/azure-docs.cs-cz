# <a name="overview-of-the-azure-security-benchmark"></a>Přehled srovnávacího testu zabezpečení Azure

Azure Security Benchmark obsahuje doporučení, která vám pomůžou zlepšit zabezpečení vašich aplikací a dat v Azure.

Tento benchmark se zaměřuje na řídicí oblasti zaměřené na cloud. Tyto ovládací prvky jsou v souladu s dobře známými standardy zabezpečení, například těmi, které jsou popsány ovládacími prvky Centra pro zabezpečení Internetu (CIS) verze 7.1.

Následující ovládací prvky se používají v benchmarku zabezpečení Azure: 

- [Zabezpečení sítě](security-control-network-security.md)
- [Protokolování a monitorování](security-control-logging-monitoring.md)
- [Identita a řízení přístupu](security-control-identity-access-control.md)
- [Ochrana údajů](security-control-data-protection.md)
- [Správa ohrožení zabezpečení](security-control-vulnerability-management.md)
- [Správa inventáře a aktiv](security-control-inventory-asset-management.md)
- [Zabezpečená konfigurace](security-control-secure-configuration.md)
- [Obrana před malwarem](security-control-malware-defense.md)
- [Obnovení dat](security-control-data-recovery.md)
- [Reakce na incident](security-control-incident-response.md)
- [Penetrační testy a tzv. red team exercises](security-control-penetration-tests-red-team-exercises.md)

Můžete si také stáhnout [tabulku Excel Azure Security Benchmark v1](https://github.com/MicrosoftDocs/SecurityBenchmarks/tree/master/spreadsheets).

## <a name="azure-security-benchmark-recommendations"></a>Doporučení pro srovnávací testy zabezpečení Azure 

Každé doporučení obsahuje následující informace: 

- **Azure ID**: Azure Security Benchmark ID, které odpovídá doporučení. 
- **ID (id) CiS**: Doporučení (doporučení) referenčních hodnot CiS, která odpovídají tomuto doporučení.  
- **Odpovědnost**: Zda je zákazník nebo poskytovatel služeb (nebo obojí) odpovědný za implementaci tohoto doporučení. Povinnosti zabezpečení jsou sdíleny ve veřejném cloudu. Některé ovládací prvky zabezpečení jsou k dispozici pouze pro poskytovatele cloudových služeb, a proto je poskytovatel zodpovědný za jejich řešení. Jedná se o obecné pozorování – pro některé jednotlivé služby bude odpovědnost odlišná od toho, co je uvedeno v benchmarku zabezpečení Azure. Tyto rozdíly jsou popsány v základních doporučeních pro jednotlivé služby. 
- **Podrobnosti**: Odůvodnění doporučení a odkazy na pokyny, jak jej provést. Pokud je doporučení podporováno Službou zabezpečení Azure, budou tyto informace také uvedeny.

Uvítáme vaši podrobnou zpětnou vazbu a aktivní účast na úsilí o azure security benchmark. Pokud byste chtěli poskytnout týmu Benchmark přímý vstup, vyplňte prosím formulář na . [https://aka.ms/AzSecBenchmark](https://aka.ms/AzSecBenchmark)
